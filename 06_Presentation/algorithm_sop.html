Phase 2 · Fitness Scoring
7-Dimension Weighted Scoring Matrix
Each of the 28 candidates scored 0–10 on 7 dimensions. Weighted sum produces a Total Fitness Score (0–100). Nuno's own benchmark hardcoded at 88.5/100 for comparison.
25%
Niche Density
Counts keyword matches for MCP, GHL, AI Agents, Vibe Coding, n8n, Claude, Make in bio + top videos. 1.5pts each, max 10.
25%
Webinar Strategy
Webinar keyword + price signal = 10. Webinar only = 5. Neither = 0. Highest weight — this is a webinar launch project.
20%
Audience Overlap
min(10, subs/100K × 10). Normalizes subscriber count against 100K baseline for non-tech entrepreneur alignment.
10%
Content Format
Webinar keyword detected in top videos = 10. Otherwise = 3. Rewards tutorial and live build content formats.
10%
Offer Structure
Price signal ($, enroll, pricing) detected on website = 10. Otherwise = 0. Checks for courses, communities, SaaS.
5%
Revenue Scale
Subscribers over 50K = 10. Otherwise = 5. Proxy for business scale and ad spend capacity.
5%
Authority Score
min(10, authority_count × 2). Google search results for [name] review/testimonial as proxy for credibility.
# Weighted fitness score — scales to 0–100 score = ( D1 * 0.20 # Audience Overlap (20%) + D2 * 0.25 # Niche Density (25%) ← highest + D3 * 0.10 # Content Format (10%) + D4 * 0.25 # Webinar Strategy (25%) ← highest + D5 * 0.10 # Offer Structure (10%) + D6 * 0.05 # Revenue Scale (5%) + D7 * 0.05 # Authority Score (5%) ) * 10 # normalize to 100 # Nuno's benchmark (hardcoded for direct comparison) nuno_scores = [8.0, 10.0, 9.0, 8.5, 9.0, 7.5, 9.0] nuno_total = 88.5 # / 100
def calculate_fitness(row): # D1: Audience Overlap (20%) d1 = min(10, (row['subs'] / 100000) * 10) # D2: Niche Density (25%) niche_keys = ["mcp", "gohighlevel", "vibe", "claude", "agent", "n8n", "make"] matches = sum(1 for k in niche_keys if k in str(row['bio']).lower() or k in str(row['top_videos']).lower()) d2 = min(10, matches * 1.5) # D3: Content Format (10%) d3 = 10 if row['has_webinar_keyword'] else 3 # D4: Webinar Strategy (25%) d4 = 10 if (row['has_webinar_keyword'] and row['price_signal']) else 5 # D5: Offer Structure (10%) d5 = 10 if row['price_signal'] else 0 # D6: Revenue Scale (5%) d6 = 10 if row['subs'] > 50000 else 5 # D7: Authority Score (5%) d7 = min(10, row['authority_count'] * 2) score = (d1*.2)+(d2*.25)+(d3*.1)+(d4*.25)+(d5*.1)+(d6*.05)+(d7*.05) return pd.Series([d1, d2, d3, d4, d5, d6, d7, score * 10])

Phase 4 · Deep Intelligence
Claude Code + RivalSearchMCP — Top 3 Only
Python scraping returned empty results for most of the 28 candidates (websites blocked requests, social platforms rejected scrapers). Deep intelligence was only achievable on the Top 3 finalists using Claude Code integrated with RivalSearchMCP.
⚠ Honest methodology note — scraping limitations

The automated web scraping phase (report2.py) returned limited results for all 28 candidates. Most websites blocked requests via Cloudflare, Instagram and LinkedIn rejected scrapers programmatically, and the Meta Ad Library is JavaScript-rendered making it inaccessible via Python requests. Individual candidate cards reflect only what was publicly accessible. Real deep intelligence was only achievable on the Top 3 finalists using Claude Code + RivalSearchMCP which includes Cloudflare bypass, multi-engine search fallback, and full website traversal. This is documented honestly as part of the iterative methodology — the scraping failures are what led to discovering RivalSearchMCP as the superior tool.
