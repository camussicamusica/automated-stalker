Part 1 — Where It Started: The Algorithm

The project began with a working algorithm, published at camussicamusica.github.io/automated-stalker. This tool was built to scrape and score potential competitors for a given YouTube creator — pulling channel data, estimating subscriber ranges, analyzing topic overlap, and ranking candidates by relevance. On paper, it was solid.

The core assumption baked into that algorithm was:

The Flawed Assumption

"If someone is a YouTuber talking about marketing and AI, they are a viable competitor and probably have a business behind them."

That assumption turned out to be wrong — or at least, wildly incomplete. When the algorithm ran and returned a list of candidates for Nuno, the results looked great on the surface. But when each candidate was manually checked, the same problem appeared again and again:

✗ Organic-only creators. No Facebook ads. No sales funnel. No product.
✗ YouTube hobbyists with knowledge but no commercial operation behind them.
✗ Educators with courses but zero paid traffic — a completely different acquisition model from Nuno's.

Nuno runs Facebook and Instagram ads. He has active sales funnels. He sells high-ticket programs. His competitors need to match that profile — not just overlap on content topics. The algorithm had found the right industry. It had found the wrong type of player within it.

Part 3 — The Pivot: Manual Research + GHL as the Filter
The Reframe

Old question: "How do I write code to find Nuno's competitors?"
New question: "What do I already KNOW that narrows this down?"

The key insight was GoHighLevel. Nuno Tavares uses GoHighLevel. His audience uses GoHighLevel. GHL is central to his business model. Therefore, his most relevant competitors would likely also be in the GHL ecosystem — either as affiliates, award winners, or featured partners.

GoHighLevel runs a formal recognition program for high-performing affiliates and agencies. These people have (a) built real businesses using GHL, (b) are actively marketing those businesses, and (c) have been validated by GHL's own team. This became the new starting pool.

Step 1
GHL Award Winners List

Assembled manually from GoHighLevel's own website, partner pages, and YouTube searches for "GoHighLevel award."

Step 2
Claude Segmentation

List filtered by 3 criteria: active YouTube channel, GHL recognition/awards, and topic overlap with Nuno's content.

Step 3
Facebook Ads Library Check

Each passing candidate was manually verified at facebook.com/ads/library. Organic-only creators were removed.

Step 4
Final 3 Selected

Three candidates clearly stood above the rest — all matching Nuno's exact profile: YouTube + active FB ads + sales funnel.

Part 4 — Final Intelligence Run

With the final three competitors selected, deep intelligence was gathered using Claude Code with the RivalSearchMCP tool (web_search, map_website, research_topic) — the same GitHub repo used throughout this project as the operational environment.

RivalSearchMCP ran parallel searches across DuckDuckGo and Yahoo, scraped review sites (marksinsights.com, ippei.com, salesforge.ai, trustpilot.com), pulled social media profiles, mapped product websites, and cross-referenced pricing data from Reddit, Scribd, and third-party analytics platforms.

The output: a full intelligence report with 7 sections per competitor, pricing tables, funnel maps, social stats, GHL connection analysis, and a master comparison table. → View Report V2

Part 5 — What Would Have Been Better

If more time had been available, this is what a better algorithm looks like:

  • Start with GHL affiliate/partner lists as the seed pool, not general YouTube searches
  • Add Facebook Ads Library check via Apify actor — automated, not manual, one query per candidate
  • Add sales funnel detection: does the candidate have a VSL page? A calendar booking page? A Skool community? These are programmatically detectable.
  • Weight the scoring: YouTube subs * 0.2 + active FB ads * 0.4 + sales funnel * 0.3 + topic overlap * 0.1
  • Output a ranked shortlist of 10–15, apply manual review only to the top tier
The Bottom Line

The manual process produced accurate results but would not scale. Rebuilding the algorithm with three hard filters (YouTube ✓ + FB ads ✓ + funnel ✓) would turn a 20-hour manual process into a 20-minute automated one. That is Part 2. Or the next time there is enough time.
